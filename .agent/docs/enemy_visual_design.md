# 👹 Enemy Visual Design Document

## 1. Общая Концепция

### Стиль

**Fantasy-Realistic Canvas Art** — детализированные процедурные враги с реалистичными пропорциями и AAA-качеством рендера.

### Принципы

1. **Читаемость** — силуэт врага понятен с первого взгляда
2. **Детализация** — каждый враг должен иметь минимум 10-14 уникальных визуальных элементов
3. **Анимация** — плавные циклы ходьбы, дыхание, процедурные движения
4. **Консистентность** — все виды (Side/Front/Back) имеют одинаковый уровень детализации
5. **Производительность** — никаких `ctx.filter`, только градиенты и альфа-блендинг

## 3. Архетипы Врагов

### 💀 SKELETON — Скелет

**Концепция:** Классический нежить-воин в потертой броне

| Вид | Ключевые элементы |
| :--- | :--- |
| Side | Щит + меч, ребра, череп в шлеме, плащ |
| Front | Грудная клетка видна, оружие параллельно камере |
| Back | Позвоночник, спина щита, затылок шлема |

**Анимация:**

- Медленное покачивание при ходьбе
- Оружие слегка движется
- Глаза светятся красным (`shadowBlur`)

---

### ⛏️ SKELETON_MINER — Скелет-Шахтёр

**Концепция:** Нежить-рабочий с киркой и мешком угля

| Вид | Ключевые элементы |
| :--- | :--- |

**Цвета:**

### 🧟 FLESH — Мясной Колосс

**Концепция:** Омерзительная груда плоти с щупальцами

| Вид | Ключевые элементы (все 14) |
| :--- | :--- |

- Кровь капает постоянно

**Пропорции:**

```text
Belly: огромный, свисающий
Legs: толстые, подключены к талии
Head: МАЛЕНЬКИЙ (0.6x scale)
```

## 4. Технические Стандарты

### Обязательные методы в каждом UnitRenderer

```typescript
class ExampleUnitRenderer extends UnitRenderer {
    // Основной рендер с определением направления
    drawBody(ctx, enemy, scale, rotation): void
    
    // Виды (все 3 обязательны)
    private drawSide(ctx, s, cycle, isMoving, t, hp)
    private drawFront(ctx, s, cycle, isMoving, t, hp)
    private drawBack(ctx, s, cycle, isMoving, t, hp)
    
    // Вспомогательные методы (по необходимости)
    private drawLegSide(ctx, x, y, cycle, rotation, s, isFar)
    private drawHead(ctx, s, t)
    private drawWeapon(ctx, s, t)
    
    // Эффекты (опционально)
    drawEmissive(ctx, enemy, scale, rotation): void
}
```

---

### Анимационные переменные (стандарт)

```typescript
const sway = isMoving ? Math.sin(cycle) * 0.06 : Math.sin(t * 0.5) * 0.02;
const bounce = isMoving ? Math.abs(Math.sin(cycle * 0.5)) * 3 * s : 0;
const breathe = Math.sin(t * 1.5) * 0.08;
const pulse = 1 + breathe;
```

---

### Определение направления

**Stabilized Facing Logic (to prevent jitter):**

```typescript
// Threshold to prevent flipping when moving vertically (jitter)
private static readonly FACING_VX_EPSILON = 0.1;

if (Math.abs(vector.x) > FACING_VX_EPSILON) {
    this.lastFacingLeft = vector.x < 0;
}

// Visuals use persistent state:
const isLeft = this.lastFacingLeft;
```

**Rotation Segments (if using 4-way/8-way sprites):**

```typescript
let facing: 'DOWN' | 'UP' | 'SIDE' = 'SIDE';
const r = rotation;
if (r > -2.35 && r < -0.78) facing = 'UP';    // Back view
else if (r > 0.78 && r < 2.35) facing = 'DOWN'; // Front view
else facing = 'SIDE';                            // Side view
```

## 6. Запрещенные Практики

### ❌ НЕ делать

1. **Статичные эффекты** — всё должно двигаться или пульсировать

   ```typescript
   // ПЛОХО:
   this.drawSlimeTrail(ctx, x, y); // Static puddle
   
   // ХОРОШО:
   this.drawDrippingGore(ctx, s, t, isMoving); // Animated drips
   ```

2. **ctx.filter для hit flash** — вызывает зависание (см. также `rendering_pipeline_guidelines.md`)

   ```typescript
   // ПЛОХО:
   if (hitFlash) ctx.filter = 'brightness(2.0)';
   
   // ХОРОШО:
   if (hitFlash) ctx.globalAlpha = 0.5 + Math.sin(t * 30) * 0.5;
   ```

3. **Несвязанные ноги** — всегда подключать к талии

   ```typescript
   // ПЛОХО:
   this.drawLeg(ctx, 0, 0); // Floating leg
   
   // ХОРОШО:
   this.drawLeg(ctx, -3 * s, 6 * s); // Connected to waist
   ```

4. **Разная детализация по видам**

   ```typescript
   // ПЛОХО:
   drawFront() { /* 3 elements */ }
   drawSide() { /* 14 elements */ }
   
   // ХОРОШО:
   // All views have same element count
   ```

5. **Непропорциональные головы**

   ```typescript
   // ПЛОХО:
   this.drawHead(ctx, s * 2.0); // Too big
   
   // ХОРОШО:
   this.drawHead(ctx, s * 0.6); // Small, grotesque
   ```

6. **"Топорные" формы из прямоугольников для экипировки**

   ```typescript
   // ПЛОХО: боевое оружие из примитивов (выглядит как кувалда)
   ctx.fillRect(-2, -15, 4, 32); // Прямая палка
   ctx.lineTo(10, 10);           // Ровные углы лезвия
   
   // ХОРОШО: реалистичные изгибы лезвия и реалистичная толщина рукояти
   ctx.moveTo(1.5, -12);
   ctx.quadraticCurveTo(8, -16, 10, -10); // Изогнутые острые края
   ctx.fill();
   ```

## 7. Производительность

### Оптимизации

1. **Использовать `ctx.save/restore` локально**

   ```typescript
   ctx.save();
   ctx.translate(x, y);
   // ... drawing
   ctx.restore(); // Clean up transformations
   ```

2. **Кэшировать Visual Assets (Boss/Elite)**

   Для сложных эффектов (свечение, аура, магические снаряды) **обязательно** использовать пре-рендеренные спрайты из `Assets.ts`.

   - `fx_boss_aura` — для аур боссов
   - `fx_boss_eye` — для светящихся глаз
   - `fx_glow_red` — для магического свечения

   *Не использовать `shadowBlur` в методах `draw()`!*

3. **Группировать однотипные операции**

   ```typescript
   // Draw all dark elements together
   ctx.fillStyle = DARK_COLOR;
   ctx.fill(); // Multiple paths with same color
   ```

4. **Избегать частых `beginPath()`**

   ```typescript
   // ПЛОХО: 10 beginPath calls
   // ХОРОШО: 1 beginPath для всей фигуры
   ```

5. **Кэшировать вычисления**

   ```typescript
   const bellyPulse = 1 + Math.sin(t * 1.5) * 0.08;
   // Use bellyPulse multiple times
   ```

## 8. Чеклист качества

Перед финализацией врага проверить:

- [ ] Все 3 вида реализованы
- [ ] Минимум 10 визуальных элементов на вид
- [ ] Ноги подключены к талии (не висят в воздухе)
- [ ] Голова пропорциональна телу
- [ ] Есть анимация дыхания/покачивания
- [ ] Damage effects при hp < 50%
- [ ] Цвета соответствуют архетипу
- [ ] Нет статичных эффектов
- [ ] Нет использования `ctx.filter`
- [ ] Scale корректен (помещается на дороге)
- [ ] Hit flash работает через `globalAlpha`
