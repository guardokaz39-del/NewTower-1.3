# Safe Refactoring Workflow

// turbo-all

## Принципы

1. **Маленькие хирургические изменения** вместо масштабных переписываний
2. **Один файл — один коммит** (логически)
3. **Каждый шаг проверяем** перед переходом к следующему

---

## Шаги

### 1. Оценка масштаба

Перед тем как начать:

- Прочитай `.agents/COMMON_MISTAKES.md` — не повторяй ошибки
- Прочитай `.agents/CONVENTIONS.md` — убедись, что знаешь правила
- Оцени: **сколько файлов затрагивает изменение?**
  - 1-2 файла → можно делать сразу
  - 3-5 файлов → написать мини-план (список файлов + что именно менять)
  - 6+ файлов → СТОП: вероятно, нужен другой подход

### 2. Проверь, что уже сделано

Частая ошибка — реализовать то, что уже есть:

```powershell
# Поиск по ключевым словам
npx grep -r "searchTerm" src/ --include="*.ts" -n
```

### 3. Выполни изменения

- **Hot-loop код:** проверь cheat-sheet в `CONVENTIONS.md` §2
- **Canvas рендеринг:** проверь §3 (state management)
- **Удаление из массива:** проверь §7 (swap-and-pop)
- **Эффекты:** используй `spawnParticle()` / `spawnExplosion()`, НЕ `add()`

### 4. Билд-проверка

```powershell
npx tsc --noEmit
```

Если есть ошибки — исправь до перехода к следующему шагу.

### 5. Тесты

```powershell
npx vitest run
```

### 6. Визуальная проверка

```powershell
npm run dev
```

Проверь в браузере:

- [ ] Нет визуальных артефактов (мерцание, прозрачность)
- [ ] Враги двигаются корректно
- [ ] Эффекты смерти работают
- [ ] Взрывы не зацикливаются

### 7. Обнови документацию

Если изменение добавляет новый паттерн или выявляет новую ошибку:

- Добавь в `.agents/COMMON_MISTAKES.md`
- Обнови `.agents/CONVENTIONS.md` если изменились правила
- Обнови `.agents/ARCHITECTURE.md` если изменилась структура
